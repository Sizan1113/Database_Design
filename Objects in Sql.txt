Querying a varray.
NOTE:  we can query varray using standard methos but it returns data the way it was inserted when is not readable for non technical users.

To query a varray we need to create a pseudo table for the varray column.

Syntax:

SELECT relational_alias.column_name, pseudo_table_alias.attribute,

pseudo_table_alias.attribute

FROM relational_table relational_alias,

TABLE(relational_alias.attribute) pseudo_table_alias

WHERE condition;


example:

SELCT s.id,a.city,a.street
from sites s, TABLE(s.address) a
Where s.id  = 1;





Nested-Tables- > Nested tables are also collection datatypes ,as the name implies it stores a table inside a single cell of the relational table.

Step 1: Create object type

CREATE OR REPLACE TYPE address_type as object(
CITY VARCHAR2(32),
DISTRICT VARCHAR2(32),
COUNTRY VARCHAR2(32));
/
show errors


step2: Create nested_table using type defination suffix (table_type)

Create TABLE address_table_type AS table OF address_type;

Step 3: Using in relational table
CREATE table sites(
ID number(5),
address address_table_type)
NESTED TABLE address store as nested_address_table_type;


Inserting into nested tables
 Insert into table_name
values('attribute value',nested_table_type(
column_object_type('objectype_attribute_value','bjectype_attribute_value'),

Querying a Nested Table:
Its similar to querying a varray .

SYNTAX:

SELECT relational_alias.column_name, pseudo_table_alias.attribute,

FROM relational_table relational_alias,

TABLE(relational_alias.attribute) pseudo_table_alias

WHERE condition;


example:

SELCT s.id,a.city,a.street

UNION:
Acts as a(aUb) but returns distinct values between the datasets.
Conditions to use UNION:
--same number of columns in both datasets.
--same datatype.
--same size of datatype.

SYNTAX:
SELECT id, name from students;
UNION
SELECT id, name from employess;


UNION ALL:
Acts as a join but returns values with null values.

Syntax:
SELECT id, name from students;
UNION
SELECT id, name from employess;

INTERSECT.
--returns matching values from both datasets.

Syntax:
SELECT id, name from students;
INTERSECT
SELECT id, name from employess;


MINUS.

--depends on which table is in left.


Syntax:

Aggregate Functions

MAX(column_name);
--Selects cell with the maximum value of a column;

Syntax:
SELECT MAX(salary) as highest_salary from employees;


MIN(column_name);
--Selects cell with the minimum value of a column;

Syntax:
SELECT MIN(salary) as lowest salary from employees;

SUM(column_name);
--Returns total sum of the values of a column.

Syntax:
SELECT SUM(salary) as Total_salary from employees; 

AVG(column_name);
--Returns average of the values in a column.

Syntax:
SELECT AVG(salary) as average_salary from employees;




COUNT(*);
-- Returns number of rows in a dataset in null values inclusive.

Syntax:

SELECT
	COUNT(*)
FROM employees;

COUNT(column_name);
--Returns number of rows in a dataset without null values.

Syntax:
SELECT 
	COUNT(ID)
FROM employees;


GROUP BY :
Makes clusters of similar data points in a categorial data before filtering or where clause.

Syntax:
SELECT MAX(salary),dept
FROM employess
GROUP BY dept;

Having clause:
--Filters the data after aggregation and where clause.

Syntax:
SELECT MAX(salary),dept
FROM employees
GROUP by dept
Having  MAX(salary) > 50000;
