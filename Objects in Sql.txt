Object-Table-> Object tables ore the table created from object types to store value as object type doesnot hold value itself as it is only the structure of the object table but the object table hold the values.

CREATE object_table_name of object type;


Using in relational table.

CREATE table table_name(
attribute datatype(size),
attribute   REF  object_type
		SCOPE IS object_table_name);

Using alter
Alter table table_name
Add (address REF  object_type SCOPE IS object_table_name);

Varray-> Varray behaves like an array of same object type column like address_varray of size (3) can hold 3 different addresses.Its limitation is its of fixed size.


There are two types of varrays:
-1D varray.

Create Type  address_varray as VARRAY(2) of address_type;



-Multidimentional varray:-> Meaning varray inside a varray.
step1:Create object _type

step 2: CREATE type column_varray as Varray(size) of object_type;

Step3: CREATE type column_varray as Varray(Size) of column_varray;

Adding in relational table either by create or alter

CREATE table table_name(
attribute datatype(size),

Alter table table_name
ADD(column column_varray);


Inserting into multidimentional varray of size 3

		column_object_type('objectype_attribute_value','bjectype_attribute_value'),

	);


Querying a varray.
NOTE:  we can query varray using standard methos but it returns data the way it was inserted when is not readable for non technical users.

To query a varray we need to create a pseudo table for the varray column.

Syntax:

SELECT relational_alias.column_name, pseudo_table_alias.attribute,

pseudo_table_alias.attribute

FROM relational_table relational_alias,

TABLE(relational_alias.attribute) pseudo_table_alias

WHERE condition;


example:

SELCT s.id,a.city,a.street
from sites s, TABLE(s.address) a
Where s.id  = 1;





Nested-Tables- > Nested tables are also collection datatypes ,as the name implies it stores a table inside a single cell of the relational table.

Step 1: Create object type

CREATE OR REPLACE TYPE address_type as object(
CITY VARCHAR2(32),
DISTRICT VARCHAR2(32),
COUNTRY VARCHAR2(32));
/
show errors


step2: Create nested_table using type defination suffix (table_type)

Create TABLE address_table_type AS table OF address_type;

Step 3: Using in relational table
CREATE table sites(
ID number(5),
address address_table_type)
NESTED TABLE address store as nested_address_table_type;


Inserting into nested tables
 Insert into table_name
values('attribute value',nested_table_type(
column_object_type('objectype_attribute_value','bjectype_attribute_value'),
column_object_type('objectype_attribute_value','bjectype_attribute_value')
);


Querying a Nested Table:
Its similar to querying a varray .

SYNTAX:

SELECT relational_alias.column_name, pseudo_table_alias.attribute,

FROM relational_table relational_alias,

TABLE(relational_alias.attribute) pseudo_table_alias

WHERE condition;


example:

SELCT s.id,a.city,a.street
from sites s, TABLE(s.address) a
Where s.id  = 1;
